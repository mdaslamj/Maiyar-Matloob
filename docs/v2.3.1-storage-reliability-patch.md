## Issue 1 — Atomic Assessment Persistence

### Root cause

`saveAssessment()` performed two independent PostgREST calls:

1. `INSERT INTO reports`
2. `INSERT INTO assessments`

If step 2 failed (invalid UUID, FK violation, network error), step 1 was already committed, leaving an orphan `reports` row with no linked assessment.

### Solution

Added PostgreSQL RPC `persist_assessment_submission` (migration `004`) that inserts report and assessment inside a single PL/pgSQL function. PostgreSQL rolls back the entire function on any failure, so no orphan reports are possible.

The Supabase storage adapter now calls:

```javascript
supabase.rpc("persist_assessment_submission", { p_report, p_assessment })
```

Community sync remains a separate step after successful persistence (unchanged from Firebase behaviour).

### Validation

| Check | Result |
|-------|--------|
| Migration defines transactional RPC | PASS |
| Adapter uses RPC, not separate inserts | PASS |
| Failed assessment insert rolls back report (PostgreSQL function semantics) | PASS (by design) |
| Community sync only runs after RPC success | PASS (code order unchanged) |

**Simulated scenario (report succeeds / assessment fails):** With the RPC, both inserts share one transaction — neither row is committed. Community sync is not invoked because the RPC throws first.

Live rollback verification requires migration `004` applied on a staging Supabase project.

## Issue 2 — Null Participant Handling

### Root cause

`saveAssessment({ participant: null })` called `resolveParticipantStorageId(null)` before the readiness guard. Accessing properties on `null` threw `TypeError`, bypassing the intended `assessment-sync-skipped` path.

### Solution

1. Guard `!participant` before calling `resolveParticipantStorageId()`.
2. Return `{ success: false, reason: "assessment-sync-skipped" }` when participant is null or has no resolvable ID.
3. Hardened `resolveParticipantStorageId()` to return `null` for non-object inputs.

### Validation

| Check | Result |
|-------|--------|
| `saveAssessment({ participant: null })` returns `assessment-sync-skipped` | PASS |
| No exception thrown | PASS |
| `resolveParticipantStorageId(null) === null` | PASS |
