# Version 2.1 — Admin Live Widget: Total Participants

Phase 1 connects **one** dashboard widget to Firestore.

## Widget

| Label | Source | Status |
|-------|--------|--------|
| Total Participants | `participants/` document count | Live when `USE_FIREBASE: true` |

All other dashboard cards remain sample data.

## Feature flag

| Flag | Behavior |
|------|----------|
| `USE_FIREBASE: false` | Sample `summary.totalParticipants` |
| `USE_FIREBASE: true` | Live count via `admin-firestore-service.js` |

`USE_FIRESTORE` remains unrelated (local storage adapter only).

## Architecture

```
admin-dashboard.js
        ↓
admin-dashboard-data.js
        ↓
admin-firestore-service.js → loadTotalParticipantsWidgetData()
        ↓
Firestore: participants/
```

UI components never query Firestore directly.

## Firestore query

Primary (preferred):

```javascript
getCountFromServer(collection(db, "participants"))
```

Fallback (documented temporary):

```javascript
getDocs(collection(db, "participants")) → snapshot.size
```

Used only if `getCountFromServer` is unavailable in the runtime SDK import.

## What is counted

- Documents in `participants/` created when a participant profile is synced (identify or assessment submit)
- Not assessments
- Not anonymous auth sessions without a synced profile document

## Security rules

Admins require read access to count participants:

```
allow read: if (own uid) || isGoogleAdmin();
```

Deploy updated rules:

```bash
firebase deploy --only firestore:rules
```

## Indexes

No composite index required for full-collection count aggregation.

## Widget states

| Status | Display |
|--------|---------|
| `loading` | Value: `Loading...` |
| `success` | Count + `Last updated: {timestamp}` |
| `empty` | `0` + `No participants yet.` |
| `permission-denied` | `—` + `Permission denied.` |
| `unavailable` | `—` + `Firestore unavailable.` |
| `error` | `—` + generic message |

## Performance

- Prefer `getCountFromServer` — server-side aggregation, no document downloads
- Fallback scans all participant documents — acceptable for early scale only; replace when aggregation is confirmed
