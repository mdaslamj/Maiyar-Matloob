# Version 2.0 — Firebase Foundation (Phase 1)

Phase 1 adds a production-ready Firebase foundation **without changing** the student UI, wording, questionnaire logic, scoring, or report generation.

## Architecture principle (locked)

| Layer | Role |
|-------|------|
| **Local storage** | User continuity and offline resilience |
| **Firestore** | Production persistence and community implementation insights |
| **Trend Engine** | Intelligence layer |
| **Admin Dashboard** | Consumer of aggregated implementation data (Phase 2 wiring) |

## Submission flow

```
Questionnaire
     ↓
Report generation (unchanged)
     ↓
localStorage (unchanged)
     ↓
Firestore sync (additive, non-blocking)
     ↓
Community aggregation (question + section counts)
```

If Firestore is unavailable, the app continues with local storage only.

## Firebase setup

### 1. Create a Firebase project

In the [Firebase Console](https://console.firebase.google.com/):

1. Create a project (or use an existing one).
2. Enable **Authentication**:
   - **Anonymous** sign-in (participants)
   - **Google** sign-in (administrators)
3. Create a **Firestore** database (production mode recommended).
4. Register a **Web app** and copy the SDK config values.

### 2. Local configuration

```bash
cp src/firebase/firebase-config.local.example.js src/firebase/firebase-config.local.js
```

Replace all `REPLACE_WITH_*` values and set authorized admin emails:

```javascript
export const ADMIN_ALLOWED_EMAILS = [
  "you@yourdomain.com"
];
```

`firebase-config.local.js` is gitignored.

### 3. Enable Firebase in the app

In `src/shared/feature-flags.js`:

```javascript
USE_FIREBASE: true
```

Keep `USE_FIRESTORE: false` during Phase 1. Local storage remains the session adapter.

### 4. Deploy security rules and indexes

```bash
firebase deploy --only firestore:rules,firestore:indexes
```

Requires Firebase CLI and `firebase init firestore` pointing to:

- Rules: `firestore.rules`
- Indexes: `firestore.indexes.json`

### 5. Seed system config

Create document `system/config`:

```json
{
  "schemaVersion": 1,
  "adminEmails": ["you@yourdomain.com"],
  "minAppVersion": "2.0.0"
}
```

This must match `ADMIN_ALLOWED_EMAILS` in `firebase-config.local.js` and the Firestore rules `isGoogleAdmin()` check.

## Authentication

### Participants (student app)

| Step | Behavior |
|------|----------|
| App load | Firebase initializes when `USE_FIREBASE` is true |
| Silent auth | `signInAnonymously()` during `application.initializeInfrastructure()` |
| UX | No login screen, no popup, no wording changes |

Implementation: `src/firebase/firebase-auth-service.js` → `ensureParticipantAnonymousAuth()`

### Administrators (`/admin`)

| Step | Behavior |
|------|----------|
| Open admin | Firebase must be configured |
| Sign in | Google popup for authorized emails only |
| Unauthorized | Access denied message; dashboard not rendered |

Implementation: `src/admin/admin-auth-gate.js`

## Security rules summary

| Collection | Participant (anonymous) | Admin (Google) |
|------------|---------------------------|----------------|
| `participants/{uid}` | Read/write own document | Denied |
| `assessments/{id}` | Create/read own assessments | Denied |
| `community/implementation/*` | Write only (no read) | Read + write |
| `admin/**` | Denied | Read + write |
| `system/config` | Read | Write |
| `system/versions` | Read/write client heartbeat | Denied delete |

Rules file: `firestore.rules`

## Migration notes

### From pre-v2 local-only

1. Existing `localStorage` data is preserved — no migration required for Phase 1.
2. Enable Firebase only after config and rules are deployed.
3. New assessments sync to Firestore going forward; historical local snapshots are **not** backfilled automatically.
4. Participant identity (name/mobile) remains in local storage; Firestore links it to Firebase Auth `uid`.

### Optional future backfill (Phase 2+)

Use `collectCommunityAnswerRecords()` from local history snapshots and batch-upload aggregates — not included in Phase 1.

### Rollback

Set `USE_FIREBASE: false` in feature flags. The app reverts to local-only behavior immediately.

- [ ] Assessment submission creates `assessments/{id}` document

- [ ] `community/implementation/*` counts increment
- [ ] App still works when Firestore write fails (offline test)

### Admin app

- [ ] `/admin` shows sign-in when not authenticated
- [ ] Unauthorized Google account is rejected
- [ ] Authorized admin sees existing sample dashboard unchanged
- [ ] Non-configured Firebase shows setup instructions
